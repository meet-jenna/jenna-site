# Photo Library

A running index of photos available for use on the site. Drop image files into `assets/photos/` and add an entry below.

> Looking for brand/integration logos? See [`assets/logos/LOGOS.md`](../logos/LOGOS.md).

When asking an agent to use these, you can say something like:
> "Use the photos described in `assets/photos/PHOTOS.md`. Place `hero-portrait.jpg` in the hero section and `team-1.jpg` in the about block."

---

## How to add a photo

1. Drag the image file into `assets/photos/` (Cursor file explorer, or Finder).
2. Add a new entry below using the template.
3. Fill in whatever you know — leave blanks if unsure, an agent can fill them in.

### Entry template

```
### filename.ext
- **Path:** `assets/photos/filename.ext`
- **Subject:** what's in the photo (e.g. "headshot of Jenna, smiling, neutral background")
- **Orientation:** portrait / landscape / square
- **Approx dimensions:** e.g. 2400×3000
- **Suggested use:** where on the site (e.g. "hero portrait", "about section", "testimonial avatar")
- **Alt text:** short descriptive text for accessibility
- **Notes:** crop preferences, color treatment, do/don't, credit, etc.
```

## Site sections that currently want imagery

Quick reference for where photos could land on the existing site (`index.html`):

- **Hero** — the area around the iPhone mock / "conversational agents" headline
- **Caller card / preview** — `caller-card-preview.html` style block
- **Section blocks** — feature rows, testimonials, "agents platform" / "creative platform" sections
- **Footer / about** — team or brand imagery

Mention which slot you want each photo in when you assign work to an agent.
